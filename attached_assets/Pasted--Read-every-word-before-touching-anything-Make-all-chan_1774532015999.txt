PART 2 — BACKEND: NEW API ENDPOINTS
File: artifacts/api-server/src/routes/files.ts
Add these four new endpoints after the existing DELETE route.
Endpoint 1 — User proceeds despite mismatch warning:
POST /api/v1/projects/:projectId/files/:fileId/cvr-proceed
This endpoint is called when a user sees the mismatch popup and chooses to proceed anyway. It requires authentication. It accepts a JSON body with one field: reason (string, required only when contentVerificationResult is "clear_mismatch", optional for "possible_mismatch").
It does the following in order: Fetch the file row and confirm it belongs to this project. If contentVerificationResult is "clear_mismatch" and reason is missing or empty, return 400 with message "A reason is required when content clearly does not match." Update the file row: set cvrWorkflowStatus to "pending_admin_review", set cvrUserReason to the provided reason or null. Insert an activity log entry with actionType "cvr_user_proceeded", entityType "file", entityId fileId, details showing the file name, the CVR result, and the user's reason if provided. Fetch all project members with role "admin" or "project_admin". For each admin who is not the uploader, insert an activity log entry with actionType "cvr_admin_notification", entityType "file", entityId fileId, details: "File [fileName] uploaded by [uploaderName] from [uploaderCompany] is pending your review. CVR result: [result]. User reason: [reason or 'none provided']. Uploaded at: [timestamp]." Return 200 with the updated file row.
Endpoint 2 — Admin approves a flagged file:
POST /api/v1/projects/:projectId/files/:fileId/cvr-approve
Requires authentication. Requires the requesting user to have role "admin" or "project_admin" on this project, OR be super admin. Accepts JSON body with one field: reason (string, optional).
It does the following: Fetch the file row. If cvrWorkflowStatus is not "pending_admin_review" return 400 "File is not pending review." Update file row: set cvrWorkflowStatus to "admin_approved", set cvrAdminAction to provided reason or "Approved", set cvrAdminActionAt to now, set cvrAdminActionBy to req.user.userId. Insert activity log entry with actionType "cvr_admin_approved", details showing who approved it, when, and their reason. Return 200 with updated file row.
Endpoint 3 — Admin rejects a flagged file:
POST /api/v1/projects/:projectId/files/:fileId/cvr-reject
Same auth requirements as approve. Accepts JSON body with one required field: reason (string, required — admin must explain rejection).
If reason is missing return 400 "A reason is required to reject a file." Update file row: set cvrWorkflowStatus to "admin_rejected", set status to "rejected", set cvrAdminAction to reason, set cvrAdminActionAt to now, set cvrAdminActionBy to req.user.userId. Insert activity log entry with actionType "cvr_admin_rejected". Return 200 with updated file row.
Endpoint 4 — Get project CVR report data:
GET /api/v1/projects/:projectId/cvr-report
Requires authentication and project membership. Accepts optional query params: from (date string) and to (date string) for historical range. If no date params provided, returns current issues only (cvrWorkflowStatus = "pending_admin_review").
Returns a JSON object with these fields: projectId, generatedAt (now), totalFilesProcessed (count of all files in project), totalFlagged (count where contentVerificationResult is "possible_mismatch" or "clear_mismatch"), totalPendingReview (count where cvrWorkflowStatus is "pending_admin_review"), totalAdminApproved (count where cvrWorkflowStatus is "admin_approved"), totalAdminRejected (count where cvrWorkflowStatus is "admin_rejected"), issues (array of file rows where contentVerificationResult is not "match" and not null, each including id, fileName, contentVerificationResult, hashComparisonNote, cvrWorkflowStatus, cvrUserReason, cvrAdminAction, cvrAdminActionAt, cvrAdminActionBy, uploadedById, uploaderName, uploaderCompany, createdAt).
Endpoint 5 — Get platform-wide CVR health summary:
GET /api/v1/cvr-health
Requires authentication. Returns: totalFilesProcessed across all projects the user has access to, totalFlagged, totalPendingReview, totalAdminApproved, totalAdminRejected, healthStatus which is "green" if totalPendingReview is 0 and totalFlagged is 0, "amber" if totalPendingReview is greater than 0, "red" if any file has cvrWorkflowStatus "pending_admin_review" and was created more than 24 hours ago without admin action.
