Add automatic PDF text extraction on every file upload. When a file is uploaded through BIMLog and passes naming convention validation — automatically extract the text content from the file in the background and store it in the database. Do not ask the user anything. Zero user input required. This is fully automatic.
Here is exactly what to build:
Step 1 — Install the required package. Run: pnpm add pdf-parse --filter @workspace/api-server
Step 2 — Add a new column to the files table called extracted_text of type text, nullable. Run the migration immediately.
Step 3 — Update the file upload API route. After a file passes validation and is saved to the database — check if the file is a PDF based on the file extension. If it is a PDF — run pdf-parse on the file buffer to extract all text content. Store the extracted text in the extracted_text column for that file record. If the file is not a PDF — store null. Do this asynchronously so it does not slow down the upload response. The user gets their validation result immediately. The text extraction happens in the background.
Step 4 — For non-PDF files that are common in BIM projects — RVT, NWD, DWG, IFC, DXF — we cannot extract text content directly. Instead automatically build a rich metadata record from the file name alone using the active naming convention. Parse the file name against the convention fields and store a JSON metadata object containing every field value decoded from the file name — for example discipline MEP becomes Mechanical Engineering, level L2 becomes Floor 2, type SP becomes Specification, originator DDS becomes the company name from the project team. Store this decoded metadata in a new column called file_metadata of type jsonb.
Step 5 — Update the AI query capability. When the AI Report Assistant receives a question about project documents — search both the extracted_text column for PDF content matches and the file_metadata column for BIM file attribute matches. Return relevant file references with the file name, uploader, upload date, and the matching content or metadata.
Step 6 — Add a new API endpoint GET /api/v1/projects/:projectId/documents/search that accepts a query parameter q and searches all files for that project — searching extracted_text for PDF content and file_metadata for BIM file attributes. Return matching files with relevant excerpts. This endpoint will be used by the AI assistant.
Then rebuild and redeploy. Do not change anything else in the app.
